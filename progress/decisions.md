# Decisions

- The canonical training dataset is `data/processed/rank14_from_local_raw/`.
- The raw source tree under `data/raw/` is now complete enough for the pipeline:
  - `train/`, `valid/`, and `test/` are all present
  - the raw image tree contains `8154` image files
  - `cards.csv` contains `8155` rows, with one extra non-image path: `train/ace of clubs/output`
- The project still trains on the normalized 14-rank dataset rather than the raw 53-label folder structure.
- Suit-specific labels are collapsed to ranks as:
  - `joker -> joker`
  - `<rank> of <suit> -> <rank>`
- The processed 14-rank dataset is materialized with hard links so the pipeline has a clean folder layout without duplicating image payloads.
- Strict stratified 5-fold cross-validation remains a non-negotiable requirement.
- The current canonical result of record is:
  - run name: `stage8_refresh_resnet50_ft_linear_lr1e4_e3`
  - backbone: pretrained `resnet50`
  - strategy: full fine-tuning
  - head: linear
  - optimizer: `AdamW`
  - LR: `1e-4`
  - epochs per fold: `3`
  - mean validation accuracy: `0.9619813205 +/- 0.0067484664`
- Stage 6, Stage 7, and the notebook/report should follow the strongest canonical full-dataset run automatically rather than assuming an older baseline.
- The notebook builder of record is `scripts/build_stage9_notebook.py`.
- The Stage 7 feature-map workflow uses the best saved checkpoint from the selected canonical run and chooses at least 2 face-card and 2 number-card examples from correct validation predictions.
- Environment constraints remain:
  - CPU execution
  - `num_workers=0` on this Windows machine because multiprocessing data loading currently fails with `PermissionError: [WinError 5] Access is denied`
